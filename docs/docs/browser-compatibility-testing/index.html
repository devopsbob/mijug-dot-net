Browser Compatibility Testing

This document describes the browser compatibility testing system for MIJUG .NET, which validates browser version detection and ensures the browserslist configuration is being respected.

Overview

The browser compatibility test suite simulates different browser versions to verify:

  1. Browser Version Detection: Accurate detection of browser name and version
  2. Browser Age Calculation: Appropriate age display with warnings for outdated browsers
  3. Browserslist Compliance: Respect for the project‚Äôs browserslist configuration
  4. User Experience: Proper warning display for unsupported browsers

ES5 vs ES6 (ES2015) Support

This project needs to be explicit about whether a browser is expected to run modern ES6 (ES2015+) code natively or should be treated as ES5-only and therefore require transpilation/polyfills. The guidance below is conservative and focused on major browser releases ‚Äî some specific ES2015 features (modules, proxies, certain APIs) may require newer minor/major releases.

  • ES6-capable (generally safe for ES2015 syntax/features):
    • Chrome: 49+
    • Firefox: 45+ (recommend 54+ for broader feature coverage)
    • Edge: 15+ (EdgeHTML 15 has broad ES6 support) and all Chromium-based Edge (79+)
    • Safari: 10+
    • Opera: 36+
    • Android Chrome / WebView: 49+
    • iOS Safari: 10+
  • ES5-only or Legacy (require transpilation and polyfills to run ES6 code reliably):
    • Internet Explorer: IE11 and older ‚Äî ES5 only; do not assume ES6 support
    • Older mobile browsers: legacy Android Browser (pre-Chrome WebView), older embedded browsers
    • Older desktop versions: Safari < 10, Chrome < 49, Firefox < 45

Notes:

  • If your code uses very new language features (optional chaining, nullish coalescing, top-level await, native modules), target newer minimum versions (for example: Chrome 61+, Firefox 60+, Safari 11+).
  • The project‚Äôs browserslist + build tooling should be used to drive transpilation (Babel / TypeScript) and polyfill inclusion. For public-facing sites that must support IE11, configure the toolchain to emit ES5 and include polyfills. For modern-only deployments, you can target ES6/ES2015 and reduce transpilation.

Running the Tests

Quick Test Commands

# Run all browser compatibility tests
npm run test:browser-compat

# Run in headed mode (visible browser)
npm run test:browser-compat:headed

# Force rebuild site before testing
npm run test:browser-compat:build

# Run specific test pattern
npx playwright test e2e/browser-compatibility.spec.ts --grep "Chrome"

Manual Test Script

# Run the custom test script with options
node scripts/test-browser-compatibility.js
node scripts/test-browser-compatibility.js --build --headed

Individual Test Cases

# Test modern browser detection
npx playwright test --grep "should detect modern Chrome"

# Test old browser warnings
npx playwright test --grep "should warn about unsupported"

# Test future browser handling
npx playwright test --grep "should handle very new browser"

# Test browserslist compliance
npx playwright test --grep "Browserslist Configuration"
